Wrong IP Address Configuration

üìãProblem Definition

Wrong IP address configurations prevent devices from communicating properly within the network, causing connectivity issues, routing problems, and inability to access network resources.

üîçSymptoms

  • Device cannot ping other devices in the same network
  • Unable to access network resources or services
  • "Destination host unreachable" error messages
  • Network applications timing out or failing to connect
  • IP conflict warnings appearing on screen
  • Intermittent connectivity issues
  • Default gateway unreachable

‚ö†Potential Causes

  • Incorrect IP address assignment outside valid subnet range
  • Wrong subnet mask configuration
  • IP address conflicts with other devices
  • Typos in IP address configuration
  • Misunderstanding of network addressing scheme
  • Incorrect VLAN IP address assignment
  • Wrong default gateway configuration

üîßSolution Steps

  1. Verify current IP configuration: ipconfig or show ip interface brief
  2. Check network documentation for correct IP scheme
  3. Verify subnet mask matches network requirements
  4. Test connectivity within same subnet: ping [neighbor-ip]
  5. Correct IP address configuration
  6. Verify default gateway is in same subnet
  7. Update DHCP reservation if using DHCP
  8. Test connectivity after IP correction
  9. Document corrected IP configuration

VLAN Misconfiguration

üìãProblem Definition

VLAN misconfiguration issues prevent proper network segmentation and inter-VLAN communication, causing devices in different VLANs to be unable to communicate or devices in the same VLAN to be improperly isolated.

üîçSymptoms

  • Devices in the same VLAN cannot communicate
  • Inter-VLAN routing not working properly
  • Broadcast traffic not reaching intended devices
  • Network segmentation not functioning as expected
  • Trunk links showing VLAN mismatch errors
  • Some ports not forwarding traffic correctly
  • VLAN database inconsistencies across switches

‚ö†Potential Causes

  • Incorrect VLAN assignment on switch ports
  • Missing VLAN creation on switches
  • Trunk port configuration errors
  • Native VLAN mismatch on trunk links
  • Inter-VLAN routing misconfiguration
  • VLAN database synchronization issues
  • Incorrect access port configuration
  • VLAN pruning blocking required VLANs

üîßSolution Steps

  1. Check VLAN configuration: show vlan brief
  2. Verify port VLAN assignments: show interfaces switchport
  3. Check trunk configuration: show interfaces trunk
  4. Create missing VLANs: vlan [vlan-id]
  5. Correct port VLAN assignments: switchport access vlan [vlan-id]
  6. Configure trunk ports properly: switchport mode trunk
  7. Verify native VLAN consistency on trunks
  8. Configure inter-VLAN routing if needed
  9. Test connectivity within and between VLANs
  10. Save configuration changes

Routing Complex RIP

üìãProblem Definition

Complex RIP routing issues occur when routers fail to agree on the best paths through the network, leading to routing loops, suboptimal paths, or unreachable destinations in RIP-enabled networks.

üîçSymptoms

  • Routes not appearing in routing table
  • Slow network convergence after topology changes
  • Routing loops causing packet circulation
  • Count-to-infinity problems with hop counts
  • Intermittent reachability to remote networks
  • RIP advertisements not being received
  • Suboptimal path selection despite better routes available

‚ö†Potential Causes

  • RIP routing protocol misconfiguration or version mismatch
  • Network topology changes causing slow convergence
  • Routing loops due to inadequate loop prevention
  • Metric inconsistencies between routing protocols
  • Timer mismatches in RIP configuration
  • Network instability causing route flapping
  • Authentication failures between RIP peers

üîßSolution Steps

  1. Check routing table: show ip route
  2. Verify routing protocol status: show ip ospf neighbors
  3. Analyze routing protocol configuration: show running-config | section router
  4. Check for routing loops: traceroute [destination]
  5. Verify network connectivity between routers
  6. Correct routing protocol configuration inconsistencies
  7. Adjust timers if necessary for faster convergence
  8. Clear routing protocol processes: clear ip route *
  9. Monitor convergence and verify stable routing

Router Interface Down

üìãProblem Definition

Router interface down issues prevent network connectivity through specific router ports, causing network segmentation and communication failures between different network segments.

üîçSymptoms

  • Interface status showing "down/down" or "administratively down"
  • No link lights on router interface
  • Connected devices cannot reach the router
  • Routing table missing directly connected networks
  • High error counts on interface statistics
  • CRC errors or input/output drops
  • Network segment completely isolated

‚ö†Potential Causes

  • Interface administratively shut down
  • Physical cable disconnection or damage
  • Port configuration mismatch (speed/duplex)
  • Hardware failure on router interface
  • Power issues affecting the interface
  • Software or driver issues
  • Network congestion causing interface errors

üîßSolution Steps

  1. Check interface status: show interfaces [interface]
  2. Verify physical connectivity and cable integrity
  3. Enable interface if shut down: no shutdown
  4. Check for interface errors: show interfaces summary
  5. Verify speed and duplex settings match connected device
  6. Reset interface configuration if needed
  7. Test with different cable or port if available
  8. Monitor interface statistics for stability
  9. Verify routing and connectivity after interface restoration

DHCP Server Failure

üìãProblem Definition

DHCP server failures prevent client devices from automatically obtaining IP addresses, subnet masks, default gateways, and DNS server information, resulting in network connectivity issues.

üîçSymptoms

  • Clients receiving APIPA addresses (169.254.x.x)
  • "Unable to obtain IP address" error messages
  • New devices cannot join the network
  • Existing DHCP leases expiring without renewal
  • DHCP DISCOVER messages not receiving replies
  • Network configuration showing "No network access"
  • Manual IP configuration required for connectivity

‚ö†Potential Causes

  • DHCP server service stopped or crashed
  • DHCP pool exhaustion (no available IP addresses)
  • Network connectivity issues to DHCP server
  • DHCP scope misconfiguration
  • Firewall blocking DHCP traffic (ports 67/68)
  • DHCP relay configuration problems
  • Database corruption on DHCP server
  • Hardware failure on DHCP server

üîßSolution Steps

  1. Check DHCP server service status
  2. Verify DHCP pool configuration: show ip dhcp pool
  3. Check DHCP bindings: show ip dhcp binding
  4. Test network connectivity to DHCP server
  5. Restart DHCP service if necessary
  6. Clear DHCP bindings if pool exhausted: clear ip dhcp binding *
  7. Verify firewall rules for DHCP traffic
  8. Check DHCP relay configuration on routers
  9. Release and renew IP addresses on clients: ipconfig /release, ipconfig /renew
  10. Monitor DHCP server logs for errors

Router ARP IP Conflict

üìãProblem Definition

ARP IP conflicts occur when multiple devices claim the same IP address, causing intermittent connectivity issues, packet loss, and network instability as devices compete for the same network identity.

üîçSymptoms

  • Intermittent network connectivity
  • "IP address conflict" warning messages
  • ARP table showing inconsistent MAC addresses
  • Ping responses from unexpected MAC addresses
  • Network performance degradation
  • Some packets reaching wrong destinations
  • Duplicate IP address detection alerts

‚ö†Potential Causes

  • Duplicate static IP address assignments
  • DHCP server assigning already-used IP addresses
  • Network device with corrupted ARP table
  • Rogue DHCP server on the network
  • Network interface cloning or virtualization issues
  • Incorrect subnet mask causing IP overlap
  • ARP spoofing or malicious network activity

üîßSolution Steps

  1. Identify conflicting devices: show arp or arp -a
  2. Check for duplicate IP assignments in DHCP scope
  3. Clear ARP table: clear arp or arp -d *
  4. Identify devices using conflicting IP addresses
  5. Change IP address on one of the conflicting devices
  6. Update DHCP reservations to prevent future conflicts
  7. Verify network subnet configuration
  8. Scan for rogue DHCP servers on the network
  9. Monitor ARP table for stability after resolution
  10. Implement ARP security measures if needed

Spanning Tree Protocol Issue

üìãProblem Definition

Spanning Tree Protocol (STP) issues can cause network loops, broadcast storms, or unnecessary blocking of links, leading to network instability, poor performance, or complete network failure.

üîçSymptoms

  • Network broadcast storms consuming bandwidth
  • Slow network convergence after topology changes
  • Ports stuck in blocking or learning state
  • Unexpected path selection in redundant networks
  • Network loops causing packet duplication
  • Root bridge changes causing network instability
  • High CPU utilization on switches during convergence

‚ö†Potential Causes

  • Incorrect STP root bridge selection
  • STP port states stuck in blocking or listening
  • STP convergence taking too long
  • Mismatched STP versions across switches
  • Incorrect port priorities or path costs
  • Physical network loops not detected by STP
  • BPDU guard or root guard misconfiguration
  • Unidirectional link failures

üîßSolution Steps

  1. Check STP status: show spanning-tree
  2. Verify root bridge: show spanning-tree root
  3. Check port states: show spanning-tree interface [interface]
  4. Identify blocked and forwarding ports
  5. Adjust bridge priorities if needed: spanning-tree vlan [vlan] priority [priority]
  6. Configure port priorities or costs if necessary
  7. Clear spanning-tree counters: clear spanning-tree counters
  8. Enable rapid spanning-tree if appropriate: spanning-tree mode rapid-pvst
  9. Monitor convergence time and network stability
  10. Verify no broadcast storms or loops exist

Port Security Violation

üìãProblem Definition

Port security violations occur when a switch port receives frames from MAC addresses that exceed the configured maximum or from unauthorized MAC addresses. This security feature helps prevent unauthorized access and MAC address spoofing attacks.

üîçSymptoms

  • Switch port automatically shutdown (err-disabled)
  • Security violation log messages appearing
  • Device suddenly losing network connectivity
  • Port LED turning amber/orange on switch
  • Maximum MAC address limit exceeded warnings
  • Unauthorized device access blocked
  • Port security counters incrementing

‚ö†Potential Causes

  • Maximum MAC address limit exceeded on the port
  • Unauthorized device connected to secured port
  • MAC address aging timer too short
  • Incorrect port security configuration
  • Network loops causing MAC address flapping
  • Virtual machines or containers changing MAC addresses

üîßSolution Steps

  1. Check port security status: show port-security interface [interface]
  2. Identify violation type and cause: show port-security
  3. Clear security violations: clear port-security sticky [interface]
  4. Adjust maximum MAC addresses if needed: switchport port-security maximum [number]
  5. Configure appropriate violation action: switchport port-security violation {protect|restrict|shutdown}
  6. Re-enable the port if shut down: shutdown / no shutdown
  7. Monitor port status and verify normal operation

Default Gateway Issue

üìãProblem Definition

Default gateway issues prevent devices from communicating with networks outside their local subnet. This critical networking component acts as the router interface that forwards traffic to remote networks.

üîçSymptoms

  • Cannot reach devices on remote networks
  • Internet access not working
  • Local network communication works fine
  • "Destination host unreachable" for external IPs
  • Ping to default gateway fails
  • Default route missing from routing table
  • DNS resolution fails for external domains

‚ö†Potential Causes

  • Incorrect default gateway IP address configuration
  • Default gateway device (router) is down or unreachable
  • Network interface on gateway device is disabled
  • Routing table misconfiguration on gateway device
  • DHCP server providing wrong gateway information
  • VLAN misconfiguration preventing gateway access
  • Firewall or ACL blocking gateway communication

üîßSolution Steps

  1. Verify current gateway configuration: ipconfig /all or ip route show
  2. Test connectivity to gateway: ping [gateway-ip]
  3. Check gateway device status and interface states
  4. Verify routing table on gateway device: show ip route
  5. Correct gateway IP address if misconfigured
  6. Restart network interface or DHCP client if needed
  7. Verify VLAN configuration and inter-VLAN routing
  8. Test end-to-end connectivity to remote networks

ACL Misconfiguration

üìãProblem Definition

Access Control List (ACL) misconfigurations can block legitimate traffic or allow unauthorized access. ACLs are critical security components that filter network traffic based on defined rules and criteria.

üîçSymptoms

  • Legitimate traffic being blocked unexpectedly
  • Services not accessible from certain networks
  • Connection timeouts to specific destinations
  • Asymmetric routing causing connection issues
  • Some protocols working while others fail
  • ACL hit counters not incrementing as expected
  • Security policies not being enforced properly

‚ö†Potential Causes

  • Incorrect ACL rule order (more specific rules after general ones)
  • Wrong source or destination IP addresses in ACL entries
  • Implicit deny rule blocking required traffic
  • ACL applied to wrong interface or direction
  • Wildcard mask errors in ACL statements
  • Missing permit statements for essential protocols
  • Conflicting ACL rules creating unintended behavior

üîßSolution Steps

  1. Display current ACL configuration: show access-lists
  2. Check ACL application on interfaces: show ip interface [interface]
  3. Analyze ACL hit counters: show access-lists [acl-name]
  4. Identify problematic ACL entries blocking legitimate traffic
  5. Reorder ACL entries if needed (remove and re-add)
  6. Correct IP addresses and wildcard masks in ACL statements
  7. Add explicit permit statements for required traffic
  8. Test connectivity after ACL modifications
  9. Monitor ACL hit counters to verify proper operation

IP NAT PAT Configuration Issue

üìãProblem Definition

Network Address Translation (NAT) with Port Address Translation (PAT) issues prevent internal devices from accessing external networks. NAT/PAT translates private IP addresses to public addresses for internet connectivity.

üîçSymptoms

  • Internal devices cannot access external networks
  • Internet connectivity not working from LAN
  • NAT translation table shows no entries
  • External services not reachable from internal network
  • Outbound connections timing out
  • NAT pool exhaustion messages
  • Return traffic not reaching internal devices

‚ö†Potential Causes

  • Incorrect NAT inside/outside interface designation
  • Missing or incorrect NAT access list configuration
  • Wrong NAT pool configuration or exhaustion
  • Firewall blocking NAT translations
  • Routing issues preventing NAT functionality
  • NAT translation table corruption or overflow
  • Incorrect overload (PAT) configuration

üîßSolution Steps

  1. Verify NAT configuration: show running-config | include nat
  2. Check NAT translations: show ip nat translations
  3. Verify inside/outside interface designation: show ip nat statistics
  4. Test NAT access list: show access-lists [nat-acl]
  5. Clear NAT translations if needed: clear ip nat translation *
  6. Correct NAT pool or overload configuration
  7. Verify routing for NAT traffic
  8. Test connectivity from internal to external networks
  9. Monitor NAT statistics and translation table

DNS Resolution Problem

üìãProblem Definition

DNS resolution problems prevent devices from translating domain names to IP addresses, resulting in inability to access websites and services by name while IP-based access may still work.

üîçSymptoms

  • Cannot access websites by domain name
  • "Server not found" or "DNS error" messages
  • Direct IP access works but domain names fail
  • Slow website loading or timeouts
  • Email services not working properly
  • Network applications failing to resolve hostnames
  • nslookup or dig commands returning errors

‚ö†Potential Causes

  • Incorrect DNS server configuration on client devices
  • DNS server unavailable or not responding
  • Network connectivity issues to DNS servers
  • DNS cache corruption on client or server
  • Firewall blocking DNS traffic (port 53)
  • DNS server misconfiguration or zone file errors
  • DHCP providing incorrect DNS server addresses

üîßSolution Steps

  1. Test DNS resolution: nslookup [domain] or dig [domain]
  2. Check DNS server configuration: ipconfig /all
  3. Test connectivity to DNS servers: ping [dns-server]
  4. Clear DNS cache: ipconfig /flushdns
  5. Try alternative DNS servers (8.8.8.8, 1.1.1.1)
  6. Verify firewall rules for DNS traffic
  7. Check DHCP DNS server assignments
  8. Restart DNS service on server if applicable
  9. Verify DNS zone files and records
